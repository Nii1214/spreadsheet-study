# ベストプラクティス

## データ入力のルール

- データの種類ごとに列を分ける
- 計算式は別のシートに分離する
- 元データは常に保持する

## 効率的な列の構成例

| 列名       | 説明         | データ型 |
| ---------- | ------------ | -------- |
| id         | 一意の識別子 | 数値     |
| created_at | 作成日時     | 日付     |
| updated_at | 更新日時     | 日付     |
| status     | ステータス   | 文字列   |

このような構造により、データの検索、フィルタリング、集計が容易になります。
